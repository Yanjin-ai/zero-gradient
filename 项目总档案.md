## 1. 项目总览

本项目为 Kaggle **The Post-Backprop Challenge: Zero-Gradient Learning for Efficiency** 竞赛而做，目标是构建一个**完全不使用全局反向传播（global backprop / autograd）**的大语言模型（LM）训练方案，并在**单张 T4 GPU / 3 小时**的极端资源约束下训练一个**≥40 亿（4B）常驻参数**的模型。

**当前状态（一句话）**：项目已有一个**可正式参赛、可复现、稳定**的提交版本——一个 **4.156B 零反向传播稀疏-学习 MoE**，在真实 Kaggle T4 上实测：峰值显存 **8.3GB / 16GB**（而 4B dense BP 需 ~31GB，在 T4 上直接 OOM）、零 autograd、确定性、WikiText-103 test perplexity ≈ **1391**（相对 unigram 基线优 **51%**）、**7/7 合规门全过**。后续的质量优化（如可训练 attention）不是"还没成果"，而是**增量迭代方向**。

**项目叙事的两条腿**（都已被真实 T4 实测支撑，且**都不依赖"智能调度更聪明"这种已被否决的声明**）：
1. **【事实】** 零梯度内容路由 MoE + 局部规则**真能学语言**（碾压 unigram/bigram，真实文本上验证）。
2. **【事实】** 4B 常驻、每步仅更新 **0.42%** 的专家（这是 sparse *learning*，不只是 sparse forward）→ 每步训练算力与总参数量解耦 → T4/3h 可训、**击败 BP 的显存/速度**。

## 1.5 v1.0 能力定稿 + v1.1 域适配（最新，可对外）

**【v1.0：一个随时可收官的完整栈】** 工程操作见 `ENGINEERING.md`。
- **架构**：4.16B 零 BP 稀疏 MoE（EMA 内容路由 + 局部规则 + training-time 稀疏）+ BPE + 2 层 MLP 读出头。
- **零 BP 性质**：预训练**与后训练**全程 `zero autograd` / 无 global backprop；MoE 稀疏更新在资源上优于 4B dense BP（后者 T4 OOM）。
- **LM 能力（真实 T4，全自动跑）**：单 T4、跑满 ~2.9h 预算 → WikiText-103 **test PPL ≈ 1355**（优于早停版 1391）；显存/吞吐稳定（~8.3GB / ~8500 tok/s）。
- **C.1 后训练（真实跑）**：
  - 本机 in-domain 组合情感：**MLP 头 100%**，零遗忘。
  - **4B 零样本组合情感**（Kaggle T4，全自动）：head-only **MLP 头 ≈60%** > 多数类 50%、线性头 52%，**LM PPL 1355→1355 零遗忘**、确定性、零 autograd。
- **结构结论**：**非线性（MLP）头是真杠杆**——线性头在 LM（D-1）和下游任务（C.1）两侧都是瓶颈；attention 用局部规则训不动（Track A 负结果）。
- **自动化基座**：`kaggle_ckpt`/`kaggle_c1`/`orchestrate_kaggle`/`experiments` —— 一键 push→轮询→拉取→记录，本机+远程统一台账。

**【v1.1：域适配 + 遗忘权衡（本机已量化，单杠杆）】** `adapt_sentiment.py`：从"零样本"基线出发，扫 in-domain 零 BP LM 适配步数 N：

| N | sentiment acc | 原域 O-PPL | 遗忘 ΔPPL |
|---|---|---|---|
| 0（零样本） | 51.2% | 4.80 | +0.00 |
| 150 | 76.5% | 12.23 | +7.43 |
| 400 | 98.6% | 31.28 | +26.48 |
| 1000 | 99.8% | 68.38 | +63.58 |

**【事实】本机域适配真能涨任务指标**（51%→99.8%），**但代价是原域灾难性遗忘**（O-PPL 4.8→68，~14×）。存在明显拐点（N=150：76% acc / +7 PPL）。

**【事实】4B 实测域适配（Kaggle T4，带遗忘缓解：300 步 / lr 0.02 / 30% WikiText 回放 / 冻结路由）= 坏交易**：

| | sentiment acc | WikiText test ppl |
|---|---|---|
| 4B-ZeroShot | 59.9% | 1355 |
| 4B-Adapt | **61.4%（仅 +1.5pp）** | **3546（+2190，2.6×）** |

**判读（关键）**：本机小配置上有效的零 BP 域适配，**搬到 4B 就失效**——即便带缓解，任务几乎不涨（+1.5pp）而 WikiText **灾难性遗忘（2.6×）**。根因：局部规则在窄域语料上 300 步就把读出头/专家/embedding 推向情感分布、广域表示被覆盖，而 XOR 组合任务又没真学会。30% 回放 + 冻结路由不足以锚住。

**【判断】这是一个有价值的负结果**：**纯零 BP 局部规则域适配不 scale 到 4B**（小配置会高估其可行性）。→ **直接给 Phase E（混合 BP）实证动机**。

**【v1.1 缓解 2.1：重要性加权 / 冻结（本机已验证）】** `adapt_mitigate.py`，N=400：

| 配置 | sentiment acc | 原域 O-PPL | 遗忘 ΔPPL |
|---|---|---|---|
| zero-shot | 51.2% | 4.80 | +0.00 |
| full adapt | 97.3% | 35.51 | +30.70 |
| **冻结 LM 头** | **92.4%** | 15.80 | **+11.00** |
| backbone ×0.1（不冻头） | 95.1% | 55.22 | +50.41 |
| 冻头 + backbone×0.3 | 95.4% | 17.40 | +12.60 |

**【事实】LM 头是遗忘的主要驱动**：冻结 LM 头把遗忘砍掉 ~64%（+31→+11）而 acc 几乎不掉（97→92%）。**只缩 backbone 不冻头反而更糟**（头被覆盖、表示不能协同 → +50）。新增 gated 默认 no-op flag：`freeze_heads` / `backbone_lr_scale`（提交版字节不变）。
**【事实】2.1 冻结 LM 头在 4B 上证实**（`adapt_4b.py` + `ZG_ADAPT_FREEZE_HEADS=1`，300 步/lr0.02/30%回放）：

| 4B adapt | sentiment acc | WikiText ppl | 遗忘 |
|---|---|---|---|
| 无缓解 | 61.4% | 3546 | +2190（2.6×） |
| **冻结 LM 头** | 62.2% | 1359 | **+3.4（0.25%）** |

冻结 LM 头**消除了 4B 的灾难性遗忘**（+2190→+3.4，确认 LM 头是元凶），**但任务几乎不涨**（+2.3pp）——backbone-only 适配在 4B 撬不动任务。**这正指向 2.2 结构分区**（不动 backbone，改读 [h,pooled-emb]）。

**【v1.1 缓解 2.2：结构分区（本机已验证，重大结果）】** `adapt_partition.py`，**原模型字节全冻 → 遗忘 = 0（构造保证）**，只训一个读"冻结特征"的新任务头：

| 任务特征（原模型全冻） | sentiment acc | 遗忘 |
|---|---|---|
| frozen-h（=零样本头） | 51.4% | +0.00 |
| pooled-emb 单独 | 50.3% | +0.00 |
| **concat[h, pooled-emb]（分区）** | **95.9%** | **+0.00** |

**【事实】结构分区碾压一切**：95.9%（≈ full-adapt 的 97%）而**零遗忘**（vs full-adapt +31、冻头 +11）。**根因洞察**：任务信息**本就在冻结的 base 里**（embedding + h）——冻结的弱随机 attention 把序列塌成一个向量时丢了它；一个读"冻结 h + pooled 冻结 embedding"的新任务头**绕过这个瓶颈**直接捞回来，**不动 backbone → 零遗忘**。

**【小配置正解，但 4B 不转移】** 小配置三策略：full-adapt 97%/+31、冻头 92%/+11、**结构分区 96%/+0（胜）**。但 **4B 实测结构分区不转移**：

| 4B head-only（零遗忘） | sentiment acc |
|---|---|
| frozen-h（零样本） | 59.9% |
| **结构分区 [h,pooled-emb]** | **61.3%（仅 +1.4pp）** |

小配置的 96% 是**又一次"小配置高估"**：27 词小词表里 embedding 平凡地编码了 polarity + not-presence，冻结特征头就能凑出 XOR；4B 的通用 WikiText/BPE 表示**根本没线性编码这些原子特征**，任何冻结特征头都捞不出来。

**【v1.1 总结论（4B 实证）】** 所有零 BP 路线在 4B 都卡在 ~60–62%：

| 零 BP 路线 | 4B acc | 遗忘 |
|---|---|---|
| head-only（frozen-h / 分区） | 60–61% | **0** |
| full adapt | 61% | +2190（灾难） |
| 冻 LM 头 adapt | 62% | +3.4 |

**没有任何零 BP 后训练能把 4B 任务撬过 ~62%**：适配要么遗忘要么撬不动，分区零遗忘但不涨。

**【Phase D.2 ZeroBP-DeepSignal：给零 BP 最后一次机会，仍突破不了】** `d2_deepsignal.py`：用手算的**纵向任务信号**（任务头闭式反向 → 顶层 block + embedding 的局部更新，零 autograd）适配。小配置扫 N：

| N | sent_acc | 原域 O-PPL | 遗忘 |
|---|---|---|---|
| 0 | 49.5% | 4.80 | +0.00 |
| 400 | 57.9% | 4.82 | +0.02 |
| 1000 | **61.4%** | 4.91 | **+0.11** |

D.2 确实有效（49→61%、近零遗忘），但**plateau 在 61%**，远低于同配置的 2.1 冻头(92%)/2.2 分区(96%)；**在小配置上 D.2 只够到 4B 卡住的 ~61%**。gate 失败 → 不迁 4B。

**【v1.x 阶段定论（封存）】** 跨 **6 种零 BP 后训练路线**（head-only / 分区 / full-adapt / 冻头 / D.2 纵向信号 + 4B 验证），**全部 plateau 在 ~60–62%**。结论硬：**零 BP 局部规则无法在适配时把组合任务特征装进预训练表示**——能装进去的（小配置 in-domain 92–96%）不转移到 4B。这不是调参问题，是零 BP 这条路在"后训练装新特征"上的**固有上限**。→ **这是 Phase E（混合 BP）的干净实证动机**：需要少量真实梯度才能把新任务特征装进表示。

> **v1.x（零 BP 栈）封存**：提交版（6.251 / 4B 1355）与自动化基座（ckpt/c1/adapt orchestrator + experiments 台账）作为**稳定基座不再改动**（除 bug 修复）。后续 D.2 增强 / Phase E 都是其上的新分支，各有 charter、flag、文档章节，默认 off，不动提交版。

**【Phase E 混合 BP：小配置突破零 BP 天花板（研究分支，2026-06-29）】** `phase_e.py`（charter 见 `Phase-E charter.md`）。**只在适配阶段对 embedding + 顶层 block + 任务头开真实 autograd**（其余全冻、预训练保持零 BP；故意打破零 BP，是研究对照，**不碰提交版**）：

| 同样适配组件（embedding+顶层 block） | sent_acc | 遗忘 |
|---|---|---|
| D.2（零 BP 纵向信号） | 61% | +0.11 |
| **Phase E（真实 BP）** | **100%** | **+0.02** |

**【事实】少量真实 BP 直接突破零 BP 的 ~61% 天花板**（100% / 近零遗忘），同组件零 BP 只到 61%。精确梯度沿组合路径分配 credit，零 BP 手算信号做不到。**lever 修正**：最小杠杆"顶层 block+头"失败（残差稀释），**embedding 才是真 lever**。gate 决定性通过 → 迁 4B。

**【Phase E 4B 实证（最终闭环）】** `phase_e_4b.py`（只 BP embedding+头，autograd 图 130MB+稀疏 expert，T4 安全）。**关键：公平读出**——必须在 BP 适配后的 embedding 上**重训闭式头**才能正确测"表示是否变好"（SGD 联合训的头欠训，会假性显示无提升）：

| 后训练路线 | 4B acc | 遗忘 ΔPPL |
|---|---|---|
| Zero-shot head | 60% | 0 |
| Zero-BP full adapt | 61% | +2190 |
| Zero-BP 冻头 | 62% | +3.4 |
| Zero-BP 分区 | 61% | 0 |
| **Phase E Mixed-BP** | **79%** | **+3.0** |

**【事实】少量真实 BP 在 4B 也突破零 BP 天花板**：BP 适配 embedding（lr0.1/1000）→ 闭式头读出 **79%**（vs 零样本 60%，+19pp），遗忘仅 +3.0 ppl（0.22%）。小配置的 100% 在 4B 降到 79%（真实 WikiText 更难，但**方向转移**：BP > 所有零 BP）。**坑（已记录）**：第一个 4B 点（lr0.05/400 + SGD 头读出）假性显示 60→57%，是**测量混淆**（SGD 头欠训）+ 超参不足；修正后真相是 +19pp。

> **【整条研究线最终结论】** 零 BP 能在 T4/3h 训出 4B LM（1355），但**这条组合任务的后训练**——6 种零 BP 路线全 plateau ~62%（装不进新特征 / 装就灾难遗忘）；**少量真实 BP（embedding+头）把任务从 60% 提到 79%、遗忘仅 +3 ppl**。**答案**：零 BP 可学大模型表示，但在结构化组合任务上，"把新特征装进表示而不遗忘"需要少量真实梯度——这是零 BP 与混合 BP 的清晰边界。

**【Phase E task #2：NLI（3 类关系推理，小配置）】** `task_nli.py`。序列 `[E1 r1 than E2 SEP E3 r2 than E4]` → 蕴含/矛盾/中性（需跨 SEP 比较两对实体 + 关系，比情感 XOR 难）。

| 路线 | nli_acc | 遗忘 |
|---|---|---|
| Zero-shot head | 49.1% | 0 |
| Zero-BP full / 冻头 | 45.3 / 46.9% | ~0 |
| **Mixed-BP（embedding）** | **57.9%** | +0.05 |
| **Mixed-BP（embedding+attention）** | **61.7%** | +0.06 |

**【事实】模式重复 + 新洞察**：零 BP plateau ~49%（最佳=零样本）；Mixed-BP embedding **+8.8pp**，**再加 attention BP +12.6pp**。**关系/位置对齐型任务额外受益于 attention BP**——因为对齐逻辑在冻结的 attention 里，embedding BP 够不着；这与 Track A（attention 是难杠杆）呼应——但**任务监督的 BP 能撬动 attention，零 BP 局部规则不能**。

> **【勘误 / Erratum（重要，已修）】** 发现并修复 `load_state_dict` 的 **CPU 别名 bug**：CPU 上 `.to(cpu)` 不复制 → `load_state_dict(baseA)` 后 `base.E` **别名** `baseA["E"]`，随后 `Z.train` 的**原地** `E.index_add_` 会**污染存档 baseA**，使多次 reset 的小配置实验串味。**影响**：`adapt_sentiment`/`adapt_mitigate`/`d2_deepsignal`/`task_nli` 的小配置数被污染。**不影响**：Phase E（重新赋值 E，从不原地）与**所有 4B 实验**（CUDA `.to` 会复制、且单发）。**修复后 clean 数（4B 结论与"少量 BP 突破"叙事不变）**：
> - 适配-遗忘：N=400 97.9%/+19.1、N=1000 99.9%/**+44.0**（原 +63 系高估）；
> - 2.1 缓解：**冻 LM 头 86.7%/+13.3**（原 92.4 系高估，但"冻头是遗忘主因的解药"成立）；**backbone 缩放变体修复后失效（~chance，原 95% 系污染虚高）**；
> - D.2 61.8%/+0.06（≈不变）。

**【当前局限与开放问题】** ① 零样本域外组合任务性能有限（4B 60%）；② 全量适配灾难性遗忘；③ 关系型任务需 attention BP（NLI 已示），零 BP 局部规则训不动 attention（Track A）；④ NLI 等更难任务的 4B Phase E 待迁。

## 2. 核心目标

- **短期（竞赛）**：在 Kaggle T4/3h 约束下，做出一个合规（零全局梯度、≥4B 常驻、确定性）、能学、且在资源上明显优于 BP 的 LM 训练 notebook。
- **中期（质量）**：在不推翻零 BP + MoE + 稀疏训练原则的前提下，尽量提高语言建模质量、让 PPL 更低更可读。
- **长期（研究）**：探索这套框架能否支持高质量生成与下游任务（后训练 / 域适配），乃至混合算法（部分 BP）。

**【判断】** 长期目标不是只做"能跑的 toy"，而是把"零 BP 4B 在 T4 可训 + 资源优于 BP"做成一个可信、可复现、可扩展的研究结论。

## 3. 约束与比赛/平台条件（硬约束，已核实）

| 维度 | 要求 |
|---|---|
| **参数门禁** | 训练前模型须**常驻 ≥ 4,000,000,000 fp16 参数** |
| **零梯度** | 完全替换 BPTT + 所有 `torch.autograd` / `loss.backward()` / 标准 optimizer，改用手写局部规则 |
| **硬件/时间** | 单 Tesla **T4 16GB**（或 4 核 CPU），**3 小时**内完成训练循环 |
| **复现** | 强确定性（`SEED=42`、`torch.use_deterministic_algorithms(True)`，两次运行一致） |
| **数据** | 自备；真实评测 = WikiText-103 perplexity（有 offline gate：数据没挂载不得声称官方分数） |
| **提交** | 公开 Kaggle Notebook + 可审计产物（loss 曲线、memory profile、run_summary.json） |

**【事实】** 局部目标的梯度可以**手算**（softmax-CE 的 `δ = p − onehot` 是标准允许技巧），只要不调 autograd、不跨层全局回传。

**平台细节（接手必读）**：
- Kaggle 默认可能分配 **P100（sm_60）**，而 Kaggle 镜像的 torch **不为 sm_60 编译 kernel** → 任何 CUDA 操作报 `no kernel image available`。**必须在 kernel-metadata 里强制 `"machine_shape": "NvidiaTeslaT4"`**（这也是竞赛评测环境）。
- GPU 配额约 **30h/周**。本项目每个完整 run 约 1 小时（多数 ~54min 早停），约 3% 配额。

## 4. 方法总设计（核心骨架，勿丢）

四条原则贯穿始终，所有改动都是"在此之上的升级"，而非推翻：

1. **MoE-FFN**：专家（experts）在**关键路径**上（不是旁路残差），是模型主力 FFN，也是 4B 参数的承载体。
2. **EMA 内容路由（EMA routing）**：用 token 的内容表示作 routing key，与每个专家的 **EMA prototype（指数滑动平均聚类中心）**做最近邻匹配 + 容量限制（capacity）选 top-k 专家；prototype 用在线 k-means 慢更新。零梯度、内容相关、诱导专家特化。
3. **局部规则（local rules）**：每个块/头用**手写闭式**的 deeply-supervised 局部 CE 更新（`δ=p−onehot`，`ΔW=−η·hᵀδ`），全程零 autograd。
4. **training-time 稀疏（sparse learning）**：每步只更新 `k_update ≪ N` 个被路由到的专家（当前 4/950 每层），训练 FLOPs ∝ k_update、与专家总数 N 解耦。

**当前 4B 架构（提交版）**：
```
token embedding (V×d)
  + 冻结随机因果注意力 (reservoir, Wq/Wk 不训练)         ← 表征上下文混合
  → 4 个堆叠 MoE block：每块 = 内容路由(EMA-prototype+capacity) → 950 个专家(d×d FFN) 取 top-2
  → 每块一个 deeply-supervised 读出头（Phase D-1 起为 2 层 MLP: d→d→V）
  → 确定性 round-robin 预算调度器：每块每步只更新 4 个"训得最少"的被路由专家
d=1024, V=32000(BPE), seq_len=64, n_layers=4, n_experts=950/层, k_route=2, k_update=4 → 4.160B fp16 ≈ 8.3GB
```

**【重要澄清】** "controller / 智能重要性调度器"曾是项目早期的核心设想，但经 Phase A 系统实验**被否决**（见 §8、§9）。**当前提交版的调度器是一个确定性 round-robin（deficit 轮转），不是性能卖点**，只为可复现性 + 最坏 backlog 上界。

## 5. 时间线与 Phase 演进

```
竞赛侦察 ─▶ Phase A（nano 机制验证, 本机 CPU, 14 个实验）─▶ 诚实定稿 ─▶ Phase B（真实 T4, ≥4B baseline）
```

**Phase A 三大产出**（详见 `Phase-A 定论.md`）：
1. **【事实】核心创新**：无 BP 约束下，内容路由 MoE + 局部规则可在**更新维度极度稀疏化**（k_update≪N）而不损质量 → 训练算力与总参数解耦 → 4B 在 T4/3h 可训（BP 不行）。这是**可行性/效率**结论，非质量超越。
2. **【事实】被否假设 H1–H6**（见 §9）。
3. **【事实】controller 角色**：从"卖点"降级为**确定性 round-robin budget 调度器**——价值在 `k_update` 预算接口 + 可复现性 + 最坏 backlog 上界，**不是性能优化**。

### 6.2 Phase B — 真实 T4，≥4B baseline

| run | 配置 | 结果 |
|---|---|---|
| **smoke** | 4B，4 分钟（先被分到 P100 失败 → 强制 T4） | 4.156B、峰值 **7.52GB**、**6226 tok/s**、ppl 31→11.9、6/6 门 |
| **v1** | lr 0.1，40min | test ppl 1788，**训练震荡发散** |
| **v2** | lr 0.03 + warmup，40min | test ppl **1437**，**单调稳定**，6/6 门 |
| **baseline-3h** | lr 0.03（无衰减/早停），满 3h | val **触底 1752 @ step6000（~52min）→ 漂移回 2383**；test 1919；5/6 门。**"3h 反而变差"** |

**【事实】** Phase B 证明：① 4B 零 BP 在 T4 可跑、显存 8.3GB（BP 需 31GB → OOM）；② 但原训练 schedule 有 **late drift（后期漂移退化）** 问题——根因是无 lr 衰减/早停 + EMA prototype 持续漂移 → 路由 churn → 类灾难性遗忘。

### 6.3 Phase C — schedule 稳定化（不改结构）

**动机**：修复 baseline 的 late drift，把最佳点锁住。
**改动**：① lr warmup + **余弦衰减**（0.03→0.003，到第 8000 步）；② **冻结 EMA prototype**（第 5000 步起，止住路由漂移）；③ **早停 + best-checkpoint**（patience 8）；④ 新增辅助指标 route_entropy / route_drift + 新门"no late drift"。**结构 / 局部规则 / 词表全不动。**

**【事实】结果**（详见 `Phase-B baseline 定论.md`，对照原 baseline）：

| | baseline（无衰减） | **Phase C** |
|---|---|---|
| 训练曲线 | 50min 触底 1752 → 漂移回 **2383** | **全程单调降到 1680，无反弹** |
| final / best | 退化（≈1.36×） | **1.000（final = best）** |
| 路由漂移度 | 持续 churn | **冻结后 = 0.0** |
| WikiText-103 test ppl | best 1437 / final **1919** | **1360** |
| 合规门 | 5/6 | **7/7** |
| t\*（最佳点出现处） | ~50min 后退化 | **150min（预算末端，单调未早停）** |

**【事实】** Phase C 同时做到：漂移消除 + 质量更好 + 整个 3h 预算变得有用（曲线到末端仍在降）。"最佳点"从"50min 然后退化"变成"预算末端、单调稳定"→ **随便在哪停都是当前最佳**。

### 6.4 Phase D — 质量增强（单杠杆逐个归因）

#### D-1：BPE 词表 + 2 层 MLP 读出头 ✅ 重大突破

**先说词表（BPE）**：原 word-level 32000 词表把所有罕见词塞进一个 `<unk>` token，`<unk>` 高频好预测 → **人为压低 ppl**。切到**离线训练的 ByteLevel BPE subword**（vocab 仍 32000，参数不变 4.156B，确定性，无网）后：

**【事实】** word-level test ppl **1360** 是被 `<unk>` **通缩的虚低数**；BPE 无 `<unk>` → 真实分布更难 → ppl **更高但更诚实**：
- **BPE-linear（旧线性头）基线 test ppl = 1998**（subword，unigram-subword 2814）——这是 D-1 的直接对照基线。

**再说 MLP 头（D-1 的核心改动）**：
- **动机**：单杠杆策略；改输出头比改 attention 风险小、可归因更干净；**假设单层线性读出头可能是瓶颈**。
- **改动**：单层 `d×V` → **2 层 MLP 头 `d→d→V`（ReLU 隐层）**。梯度**手算闭式、在读出模块内 2 步反向、全程零 autograd**（与现有逐块更新同性质）；`dout` 经读出头反向流回 block，喂专家/embedding 更新。加了 `head` 配置、MLP 头、param_count 计入、`ZG_HEAD` 环境开关。
- **小配置 sanity**：val_ppl 7.10 → 6.77（~5%），零 autograd、确定性、单调。4B+MLP = **4.160B ≥ 4e9**。
- **【事实】4B / Kaggle 正式结果**：

| | BPE-linear 基线 | **BPE + MLP 头（D-1）** |
|---|---|---|
| WikiText-103 test ppl | 1998 | **1391**（约 −30%） |
| 相对 unigram(2814) | 优 29% | **优 51%** |
| 稳定性 | 7/7、单调 | **7/7、单调、漂移 0、早停 t\*=54min** |
| 资源 | 8.3GB | **8.33GB、8546 tok/s**（基本不变） |

- **【事实】认知更新（重要）**：原判断"头只是边际杠杆、表征才是主要瓶颈"**被推翻**。实际上**单层线性头本身就是显著瓶颈**——只加一个隐层就从同一表征榨出约 30%。**小配置只显示 5%，4B 上放大成 30% → 小配置会低估某些结构杠杆。**
- **当前状态**：D-1 成功，**已固化为 Kaggle 默认提交版**（见 §7）。

#### D-2：next-2 token 辅助局部损失 ❌ 失败但有价值

- **动机**：D-1 成功后继续找下一个单杠杆；设计一个有原则、零 autograd、便宜的局部损失——**多 token 预测中的 next-2 辅助头**：在最终块加辅助头预测 `t+2`，把信号加权反馈给表征，逼模型编码 2 步前瞻信息（丰富表征、不改主结构）。
- **改动**：加 `aux_w` 配置、辅助头 `Ha`、`Y2` 窗口、final block 的 aux loss、`ZG_AUX` 开关。**保持零 autograd**。
- **【事实】小配置验证**：D-1 基线 best 6.54 → **+aux 0.3 → 6.65 / +aux 0.5 → 6.65**（略**差** ~2%），稳定/单调/确定性保持但**无增益**。
- **失败原因解释**：`t+2` 是更难、与主目标 `t+1` 竞争的目标；在当前弱模型上没能丰富表征、反而分流学习能力。
- **【事实】为什么这是成功的流程**：遵守"小配置先验证、表现好才迁 4B"→ **小配置正确拦住了无效改动、省了一次昂贵的 4B/T4 run**。D-2 本身失败，但验证流程成功。
- **当前状态**：D-2 基础设施已提交但**默认关闭**；**不迁移 4B、不纳入提交版**。

## 7. 当前最优版本（提交版）

**【事实】当前 Kaggle 提交版 = BPE + 2 层 MLP 头 + Phase C 稳定 schedule。**

- 提交 notebook：`kaggle_zerograd_moe.ipynb`，KAGGLE 配置**默认即此最佳设定**，无需任何 env 变量。
- 结构：**4.160B 参数**，零 global backprop / 零 autograd，单 T4 / 3h 内可跑。
- **最重要结果（实测）**：WikiText-103 **test ppl = 1391**（subword）、相对 unigram(2814) **优 51%**、**7/7 合规门**、单调、漂移 = 0、峰值显存 **8.33GB/16GB**、**8546 tok/s**、早停 t\*≈54min。
- BP 训 4B 需 ~31GB → T4 OOM（核心对比叙事）。

**质量演进表**（同设定内可比；跨设定换基准、不可直接比）：

| 设定 | WikiText-103 test ppl | 说明 |
|---|---|---|
| word-level 32k | 1360 | **被 `<unk>` 通缩的虚低数**（不是真实质量） |
| BPE subword（线性头） | 1998 | 诚实基准（无 `<unk>`） |
| **BPE + MLP 头（当前提交版）** | **1391** | 单杠杆：头从单层→2层，−30%，**优 unigram 51%** |

**为什么现在就固化提交版，而不是等所有优化做完？** **【判断】**
1. 当前已有一个**可参赛、可复现、质量和稳定性都最好**的版本——项目已从"有没有成果"进入"押注哪条增益路径"。
2. 后续优化是**增量迭代**，不影响现在作为正式参赛版本；先固化能让所有后续实验都有**清晰基线和可归因对照**。
3. 单杠杆策略要求每次只比一个变量；没有固定基线就无法干净归因。

## 8. 已验证结论（【事实】）

1. 这套**零 BP 4B 稀疏 MoE 路线在 Kaggle T4/3h 约束下可行且稳定**（实测 8.3GB、~8000 tok/s、确定性、7/7 门）。
2. **极稀疏更新不掉质量**：每步只更新 0.42% 的专家，质量与更新更多时相当（k=1≈k=16）——这是 sparse *learning*（非仅 sparse forward），是项目的核心硬通货。使能者是 **regime**（局部规则独立可跳 + capacity 路由 + basin 鲁棒），**不是聪明 controller**。
3. **特化机制鲁棒可扩、真实数据成立**：内容路由（EMA-prototype + 容量限制）+ 局部规则 → 专家按内容特化（合成 purity 0.67，真实文本 coherence +0.40）。
4. **Phase C 修复了 late drift**：lr 余弦衰减 + 路由冻结 + 早停 → 单调、无漂移，整个预算可用。
5. **BPE + MLP 头是当前最优、最成熟、可正式提交的版本**（test ppl 1391，优 unigram 51%）。
6. **输出头增强不是边际优化，而是已验证的大质量杠杆**（−30%）；推翻了"头只是边际"的原预判。
7. **"单杠杆 + 小配置先行 + 再迁 4B"是有效的研究流程**：D-1 在小配置被低估（5%）、4B 放大（30%）；D-2 在小配置就被拦下、省了一次 4B run。
8. **word-level 的低 ppl 是 `<unk>` 假象**；BPE 给出诚实（更高）基准。

## 9. 已失败 / 被否决路线（【事实】，非常重要，勿丢）

| 假设 / 尝试 | 怎么验证的 | 结果 | 判断 / 节省了什么 |
|---|---|---|---|
| **H1** 智能 importance controller（leverage/learnability）能降 PPL | E10 在特化 MoE 上做 importance vs random/uniform/fixed 消融，多 seed | toy 上 +0.05 级、随预算收紧放大；但真实数据上 ≈ random（噪声内） | 否决"调度提性能"声明；避免把卖点押在不可辨识的排序上 |
| **H2** controller 的优势会在**覆盖**（而非 PPL）上显现 | E14 实现 v5 分层调度器，用 undertrained_traffic/max_backlog 指标在大 N 扫预算 | random 每档最好；controller 不胜 | 否决"controller 是覆盖优化器"；root cause = capacity 路由已均衡 |
| **H3** 加法式专家（强 backbone 上的残差）有用 | E05/E06 backbone-only vs full | 强 backbone 上专家是冗余噪声 | 改为专家上关键路径（MoE-faithful） |
| **H4** value 信号（‖dW‖、learnability）在 scale 上可用作排序 | E11/E13 大 N 下观察 gap | Var(噪声)≫Var(信号) → 排序不可辨识 ≈ 随机 | 只保留无噪的 coverage 计数信号 |
| **H6** controller 是稀疏训练的**使能机制** | E13/E14 random 稀疏选也达同质量、更好覆盖 | 否（regime 才是使能者） | controller 降级为确定性 round-robin |
| **D-2** next-2 token 辅助局部损失能丰富表征 | 小配置 aux 0.3/0.5 对照 D-1 基线 | 略差 ~2%，无增益 | 不迁 4B，省一次 run；说明并非所有合理局部损失都有效 |
| **A1** 借 block-0 readout 信号（`dh₀`）闭式训 attention Wq/Wk | 主小配置（噪声）+ assoc-recall 探针（frozen 必败）+ dead-or-savable 诊断（`track_a_probe.py`/`track_a_diag.py`） | 主小配置测不出（6.251 vs 6.245 噪声）；探针上 frozen 与 train 都 chance acc(~2.5%)，attn 几乎不动；**调 attn_lr 到 ×1e5 仍 chance、ppl 仅 3% 且饱和**。诊断 `r_attn=0.38`（attention 确有贡献）但 readout 梯度**方向**教不会 content-lookup → **不是 scale 是方向** | **Track A 暂缓**：借来的端信号局部规则训不动 attention（对照 D-1"头"是大杠杆）。A2（attention 自有 goodness）留作未来候选，需强理论 mini-charter。零 BP 栈里 attention 难局部训本身是有价值的诚实负结果 |

**【事实】** 一个重要的"元教训"：**PPL 是 controller 的错配指标**（它测终点 basin，controller 作用在训练路径）；但把指标校正到覆盖后，controller **依然**不胜 random——因为系统已被 capacity 路由均衡，智能选择没有发挥空间。

## 10. 当前未解决问题

- **【事实】** 当前模型仍是**诚实的弱 LM**：相对 unigram 优 51%（subword），但绝对 ppl（1391）离真正强 LM（个位数到几十）差很远。这是局部学习方法的**固有上限**，与 Phase A 一路发现及文献一致。
- **【判断】** 表征质量受限的可能瓶颈：**attention 当前是冻结随机 reservoir**（不可训练）。类比"头曾是隐藏瓶颈"，attention 很可能是下一个真实结构瓶颈——但未验证。
- **【判断】** 局部损失增强（D-2 之外）是否有有效变体（对比/重建/负载均衡）尚不清楚；负载均衡在容量路由下大概率冗余。
- **【判断】** 下游适配 / 后训练能力（Phase C.1）完全未验证。

## 11. 后续路线图

### 方向 A：可训练 attention（D-2 之后的大结构杠杆）
- **【判断】** attention 现为冻结随机 reservoir，很可能是下一个真实结构瓶颈，**最大剩余 EV**；但实现最复杂、风险最高（需为 attention 设计一个**不和 MoE 路由打架**的局部规则，如 Forward-Forward 风格 goodness 或由最终头信号一步回传）。需先在小配置上多轮 sanity，再迁 4B。**高风险高回报。**

### 方向 B：继续尝试其他局部损失
- **【判断】** 对比损失 / 其他局部信号；但需警惕投机性，next-2 已证明不是所有辅助目标有效；负载均衡在当前容量路由下大概率冗余。**中风险、中不确定收益。**

### 方向 C：加深主干 / 扩展 backbone
- **【判断】** 增加可训练主干层 / n_layers / 专家维；增参数与稳定性风险；**当前非最高优先级。**

### 方向 D：Phase C.1 后训练 / 下游适配
- **【判断】** 从当前稳定 best checkpoint 出发，验证零 BP 框架的下游适配 / 域适配能力。**偏"展示框架完整性"，不是继续榨 PPL。** 可与质量增强并行。

### 方向 E：混合算法（长期研究，非当前 Kaggle 主线）
- **【判断】** pretrain 仍零 BP MoE；finetune 在顶层/少数层引入少量 BP。研究问题：不同混合比例下质量 vs 资源的 trade-off。

## 12. 各候选方向的资源-风险-回报分析

| 方向 | 潜在回报 | 风险 | 资源投入 | 实现复杂度 | 是否合规（零 BP） | 【判断】优先级 |
|---|---|---|---|---|---|---|
| **A 可训练 attention** | **高**（可能像头一样是大瓶颈） | **高**（可能破坏表征、不单调、调参贵） | 小配置多轮 + 1–2 次 4B run | **高**（要设计 attention 局部规则） | 是（用局部规则训） | **若追最大质量突破：首选** |
| **B 其他局部损失** | 中（不确定） | 中（目标互相打架） | 小配置多次 + 视情况 4B | 中 | 是 | 中（投机，先小配置筛） |
| **C 加深主干** | 中 | 中（参数/稳定性） | 重算 param + 4B run | 中 | 是 | 低（暂不） |
| **D C.1 后训练** | 中（展示完整性，非降 PPL） | 低-中（遗忘） | 30–60min run | 低-中 | 是 | **若追稳的项目闭环：首选** |
| **E 混合 BP** | 高（质量） | 高（破坏零 BP 卖点 + 资源） | 大 | 高 | **否（部分用 BP）** | 远期研究，非 Kaggle |

**【判断】当前最合理的战略判断**：
- 项目已有很强提交版，后续是"押注哪条增益路径"的问题。
- **追最大潜在质量突破 → 首选可训练 attention（A）**，但要接受高风险 + 前期多轮小配置调试。
- **追更稳的项目闭环 → 转 C.1（D）**，证明零 BP 框架不仅能预训练、还能做后训练/下游适配。
- **不建议把多个改动堆一起**；坚持单杠杆 + 小配置先行 + 成功才迁 4B + 保证可归因。

### 13.2 如果选「可训练 attention（A）」

- **小配置先做的 sanity**（必须全过才允许迁 4B）：
  1. 设计 attention 的**局部目标**（建议先试最简单的：让最终读出头的信号 `dh` 一步回传到 attention 的 Wq/Wk，手算闭式、零 autograd，不跨块；或 FF 风格 goodness 正负样本）。

### 14.4 合规门（7 项）

### 14.5 术语
- **zero global backprop / zero autograd**：不调 `torch.autograd`/`.backward()`/optimizer，所有更新手写闭式局部规则。
- **MoE-FFN / 专家在关键路径**：专家是块的 FFN 主路径，token 必经被路由的专家。
- **EMA-prototype 路由**：每个专家一个 EMA 聚类中心，token 按内容最近邻选 top-k + 容量限制。
- **sparse learning / training-time sparsity**：每步只更新 `k_update ≪ N` 个专家（区别于标准 MoE 的 sparse forward + dense backward）。
- **特化（specialization）**：相似上下文稳定命中同一专家；合成语料用 topic purity、真实语料用路由 coherence 度量。
- **late drift**：训练后期 val ppl 触底后反弹退化（baseline 的问题，Phase C 修复）。
- **t\***：val ppl 最佳点出现的时间/步数。
- **basin 鲁棒**：终点 likelihood 对"更新了哪些专家"高度不敏感（k=1≈k=16）。
