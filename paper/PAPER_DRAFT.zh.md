# 规模化的 ZeroBP 与"能力的架构性":一个无反向传播 4B 语言模型的边界研究,配以可训练注意力对照

**技术报告草稿 —— v0.1。** 目标:workshop 论文(8–12 页)。所有数字均可追溯至随附仓库的实验台账与逐次运行摘要;锁定结论处给出 commit 哈希。

---

## 摘要

本研究考察:在严格资源预算(单块 T4 GPU,3 小时)下,一个*无反向传播*的语言模型能走多远 —— 以及更重要的,*它为什么止步*。本工作构建了 **ZeroBP-4B**,一个 41.6 亿参数的内容路由专家混合(MoE)语言模型,其全部训练使用手写**局部更新规则**、**零全局反向传播**(无 `autograd`),并将其锁定为一个确定性、可复现的 Kaggle 提交(WikiText-103 测试困惑度 ≈ 1391 早停 / ≈ 1355 跑满预算)。在这个固定骨架上,本研究跨三种任务结构(探测现代 LLM 能力的 bag 式情感、关系蕴含 NLI、多步模算术)映射**能力边界**,并逐步注入反向传播(BP)。这道边界清晰且*结构性*:一点 embedding 路径 BP 把情感从 60% 提到 79%,但无论少量还是*深度* BP 都无法在 4B 上装入关系或多步结构(两者都停在 chance),三条"ZeroBP 原生"路线(更丰富数据、结构辅助目标、局部注意力规则)各加 ≤2pp。一个不塌缩读出探针表明关系结构*在表示的任何位置都不存在*。

为检验这些失败是*任务*的属性还是*架构*的属性,本研究引入 **Phase H**,一个严格隔离的对照:一个用普通全反向传播训练的标准多层可训练注意力 Transformer。把 ZeroBP 卡在 65.7% / chance 的同一批合成任务,被一个 0.8M 参数的 Phase-H 模型解到 100%;在**真实 SNLI** 上 Phase H 达到 **69.97%**,而 ZeroBP-4B 停在 chance(33.4%)。这幅图景并非一边倒,在此如实报告:Phase H 装进了浅多步推理(k≤3=100%)却撞上它*自己*的**抗规模深度墙**(从 4.7M/6k 步到 21.3M/15k 步不变);一个从零 char LM 在真实 GSM8K 上仅达 2% exact-match;并且 —— 和 ZeroBP 一样 —— 在修正一个读出坍缩 artifact 后,两栈在真实 SST-2 上都近 chance。研究中的两个工作假设在过程中被数据推翻,并在记录中纠正。最终形成一个完整的 arc:**ZeroBP 的关系/多步天花板是架构性的,不是根本性的 —— 一个可训练注意力骨架在真实关系数据和浅多步推理上跨过了它,同时暴露出它自己诚实的新上限。**

## 1. 引言

现代大语言模型以巨大的算力成本用全局反向传播训练。两个正交问题驱动本工作:

1. **在严格单 GPU 预算下,不用全局 BP 能保住多少语言能力?** 无反向传播 / 局部学习方法对内存和硬件受限场景有吸引力,但它们在规模下的能力天花板缺乏刻画。
2. **当这样的模型在某任务上失败,失败是*算法*(梯度不够)、*架构*(骨架无法表示该结构)、还是*任务本身*?** 三者通常无法分离,因为研究者一次改动了很多东西。

本工作以一个刻意分层的研究拆开二者,组织成**三条线**:**提交线**(一个干净的纯 ZeroBP 4B 基线,§3)、**边界线**(在该冻结骨架上、把它当研究对象,测量任务×方法能力矩阵,§4)、**对照线**(一个严格隔离的 Phase H 栈 —— 标准可训练注意力骨架 + 全 BP —— 用作架构对照,§5)。贡献不是新的 SOTA 数字,而是一项**受控边界研究**:一个可复现的无 BP 4B 基线、一个带干净提交/研究隔离纪律的系统能力矩阵、以及一个把"ZeroBP 做不了 NLI"变成"*这个*骨架不行、*这个*骨架行、连它*都*在哪里止步"的架构对照。

本文前置三条塑造结果的方法论承诺:(i) **公平读出** —— 能力用一个全新的、任务无关的读出头测量,因为研究发现不公平的读出能凭空制造或抹掉表面能力(§4.4、§5.5);(ii) **单杠杆改动** —— 每个实验只动一个结构自由度;(iii) **诚实纠错** —— 研究中的两个假设被后续实验推翻,报告保留纠正而非最初的猜测。

## 2. 相关背景(简)

本文的 ZeroBP 训练器属于**局部学习 / 反向传播替代**方法家族(局部损失、target propagation、forward-only 与 Hebbian 式更新)和**稀疏激活 MoE** 模型,但组合不寻常:一个大型常驻专家库,专家由手写局部规则在确定性稀疏预算下更新,配一个冻结随机注意力"reservoir"(reservoir computing 精神),把一个末位塌缩表示喂给堆叠的路由块。评测维度(LM 困惑度、情感、NLI、多步算术 / GSM8K、GLUE-SST2)覆盖了刻画现代 LLM 的标准能力维度,规模上可负担受控消融。本工作不宣称任何单一组件的新颖性;贡献是**边界方法论与架构对照**。

## 3. 提交线:一个纯 ZeroBP 4B 基线

### 3.1 动机与约束

目标环境是单块 T4(16GB),3 小时墙钟,自备数据 —— 在此标准 BP 训练 4B 不可行(实测该模型的 BP 前+反向约需 ~31GB,即 OOM)。ZeroBP 彻底绕开反向传播:没有全局损失梯度,每个参数更新都是局部规则。

### 3.2 架构与零 BP 更新

ZeroBP-4B 是一个 decoder 式语言模型,数据流(代码级事实):

```
token x ─► E[x] + pos                                       (embedding;|V|=32000, d=1024)
        ─► 冻结随机因果注意力 (Wq,Wk 不训练)                    (单层"reservoir"注意力)
        ─► h = (emb + att·emb)[:, -1]     ← 末位塌缩:整条序列 → 一个 [B,d] 向量
        ─► 4× 堆叠 MoE block:内容路由(EMA-prototype+capacity) → 950 专家 top-2
        ─► 每块深监督 2 层 MLP 读出头 + 确定性 round-robin 预算(每块每步更新 k_update 个专家)
```

参数 `d=1024, |V|=32000, seq_len=64, 4 层, 950 专家/层, k_route=2, k_update=4`,得 **41.6 亿 fp16** 常驻参数。三个性质对后文重要:

- **无全局梯度。** 训练全程 `torch.set_grad_enabled(False)`,由 gate 断言。每个专家由一条手写局部规则更新,信号来自每块深监督读出;路由是不可微的内容分配(EMA 原型 + 容量)。
- **确定性稀疏预算。** 一个 round-robin schedule 每块每步只更新 `k_update` 个专家(专家库的 0.42%),实验发现这相对稠密更新不掉 LM 质量(nano 消融里 k=1 ≈ k=16),并给出最坏 backlog 上界。一个学习式重要性 controller **不**胜此随机/round-robin schedule,被否。
- **冻结注意力 + 末位塌缩。** 单注意力层是一个冻结随机 reservoir(`Wq,Wk` 从不训练),整条序列在路由块之前被塌缩到最后一个位置。这两点成为 §4 的头号疑点。

### 3.3 Kaggle 提交:纯净性、gate、可复现

构建了一个自包含提交,嵌入当前训练器并运行**默认(纯 ZeroBP)路径**、无研究开关。完整性被强制并检查:

- 提交文件**没有** `.backward` / `autograd.grad` / `enable_grad`;所有 BP 存在于提交永不 import 的独立研究脚本;每个研究开关(`attn_train`、`save_ckpt`、`freeze_heads`、`backbone_lr_scale`、`aux_w`)默认空操作。
- 一套 gate 断言:常驻参数 ≥ 门禁、**零 autograd**、损失单调、val-ppl < unigram、**确定性(重跑一致)**、无后期漂移。小配置默认确定性复现 `final_ppl = 6.251`;完整 4B 运行装进 T4,峰值 ~8.3GB、~8546 tok/s。
- 修复了一个真实的可复现 bug:早前的提交 notebook 嵌入了一个*改进前的 word-level 快照*;从当前单源训练器重建 notebook,使提交反映已文档化的 BPE+MLP 配置。

**基线结果。** BPE 子词 + 2 层 MLP 读出头 + 带路由冻结的余弦 schedule("Phase C"),ZeroBP-4B 达到 **WikiText-103 测试困惑度 ≈ 1391**(早停,t\* ≈ 54 分钟)/ **≈ 1355**(跑满 ~2.9 小时预算),优于早前 word-level 阶段(≈ 1360),并比 unigram 基线好约 51%。2 层 MLP 头是一个大杠杆;朴素线性头弱得多。这给出边界研究一个干净、确定、无 BP 的 4B 对象。

**工程勘误(与 §4 相关)。** 发现并修复了一个 `load_state_dict` CPU 别名 bug(clone-on-load),它曾污染部分*多 reset 小配置*实验(reset 后的 base 别名了 golden checkpoint,随后被原地污染)。下文所有小配置数为修正后值;**4B 与单发实验不受影响**(CUDA 复制 + 单发隔离),故 79% 突破与 4B 矩阵行不变。这也是将小配置结果视为*指示性*、并在锁定能力论断前要求 4B 确认的原因。

## 4. 边界线:一个 ZeroBP 能力矩阵(Phase E/F/G)

现在把 ZeroBP 骨架固定,追问后训练能装入什么,只向 **embedding / attention / 任务头**注入 BP(从不改架构骨架)。所有准确率用**公平读出**:在*冻结*适配后表示上训一个全新闭式(无 autograd)分类头,故所测的是*表示*是否获得结构,而非 BP 头是否过拟合。

### 4.1 三种任务结构

- **情感** —— 一个 bag/组合标签(顺序不敏感的词级组合)。
- **NLI** —— 一个 3 类关系蕴含,需要跨句对齐*两个*实体对及其关系(蕴含 / 矛盾 / 中性)。
- **两步算术** —— 一个 5 类真正顺序的 fold `((d1 ∘ d2) ∘ d3) mod 5`。

### 4.2 任务 × 方法矩阵

| 任务 | 结构 | 4B zero-shot | 4B 少量 BP | 小配置最好 |
|---|---|---|---|---|
| 情感 | bag 组合 | 60%(59.9) | **79%**(embedding + 头,公平读出) | 100% |
| NLI | 关系对齐 | 33.4%(chance) | 33.4%(**不**转移) | 62% → 65.7%(深 BP) |
| 两步算术 | 多步计算 | 24.7%(小) | ~19–21%(chance) | ~chance(gate 失败) |

关键锁定结果:

- **情感突破(79%)。** 4B head-only 后训练达 59.9%(MLP 头)且*零遗忘*(LM ppl 1355.4 → 1355.4)。加一点 embedding 路径 BP 提到 **79%**,代价是 +3.0 ppl 遗忘(公平闭式读出)。消融显示 **embedding 是杠杆**(embedding-only ≈ embedding+top-block)。分析还追出一个**测量混淆**:一个早期"57%"点用了欠训的 SGD 读出头;公平闭式头修正为 79% —— 这是读出选择能制造/掩盖能力的第一个迹象。
- **NLI 不转移。** 在 4B,zero-shot / emb-BP / emb+attn-BP **全是 chance(33.4%)**;大预算重试(3000 步,attn lr 0.5)仍 chance,故这是结构限制而非欠训。
- **多步不可安装。** 即便小配置,两步任务在任何 BP 组件下都过不了 gate(~chance);不带到 4B。

### 4.3 三条 ZeroBP 原生结构杠杆(Phase F)

ZeroBP 能否*不*跨入 BP 就装入关系结构?三条路线:

- **F1 —— 更丰富预训练数据**(一致关系对 + QA):NLI zero-shot 49.1% → **51.3%(+2.1pp)**。数据单独不够。
- **F2 —— 结构辅助目标**喂给 ZeroBP 局部规则:51.3% → **51.8%(+0.5pp)**。ZeroBP 即便被交给结构目标也刻不出关系几何。
- **Track A —— 局部注意力规则**(从读出信号反馈到 `Wq,Wk`):形不成有用对齐。

相较之下,一点 BP 在小配置里*部分*装入(公平新鲜头 NLI 51.3% → **58.8%**(embedding),→ 59.1% 加 attention)—— 但 attention 的额外贡献仅 **+0.3pp**,embedding 主导,且**不转移到 4B**。墙的本质是 **BP vs ZeroBP**,而在 BP 内部是 **embedding**、不是 attention,扛起了(有限的)信号。

### 4.4 关系结构在哪里?(Phase G,v2.0 探针)

开辟了一个 research-only 的"v2.0"空间,在骨架上测三个结构杠杆,每个单杠杆、永不触碰提交:

- **不塌缩读出**(`v2_readout`)。在*同一冻结* base 上,把末位塌缩换成 mean-pool / all-positions / concat。结果:NLI **不**升 —— 反而*降*(last-h 50.4%、mean-pool 34.9% ≈ chance、all-positions 32.9% ≈ chance、concat 47.5%)。**关系结构不在冻结表示的任何位置**;更好的读出捞不出不存在的东西。这*推翻*了"末位塌缩是瓶颈"这一假设。
- **可训练 attention,隔离**(`v2_attn`)。冻结 embedding,*只*在 NLI CE 上训 `Wq,Wk`,公平新鲜头:**51.3% → 51.3%(+0.0pp)**。已验证权重确实移动(‖ΔWq‖ ≈ 0.080、‖ΔWk‖ ≈ 0.079)且 embedding 冻结(‖ΔE‖ = 0)。故 emb+attn 的 59.1% *全部来自 embedding*;在此骨架上可训练 attention *单独*不是杠杆。这*推翻*了更新后的"可训练 attention 是真杠杆"假设。
- **更深 BP**(`v2_deepbp`)。让 MoE 块本身参与 BP。NLI(小):floor 49.1 → emb 57.9 → emb+top-block 64.3 → emb+all-blocks 63.9 → **emb+all+attn 65.7%**(已验两块各训 ~22–23/48 专家)。算术:*每一*BP 深度臂都停在 chance(19–21%)。故深度在*小配置*给出真实关系信号,但**不转移 4B**(NLI-4B 仍 chance),多步在每个深度都死。

### 4.5 边界结论(ADR-002)

证据支持 ZeroBP-4B 骨架一道清晰、结构性的边界:

> 对 **bag 式**任务(情感),ZeroBP + 一点 embedding BP 有效(4B 79%)。对**关系**(NLI)和**多步**(算术)任务 —— 现代 LLM 核心能力 —— 当前 ZeroBP-4B 架构与训练范式无法装入结构:关系几何*在表示的每个位置都不存在*(读出捞不出)、可训练 attention 单独无用、多步在任何 BP 深度不可安装。决定因素是**任务有多少落在 embedding**(BP 可装)vs 多少需要*通过冻结块的顺序计算*或*跨句注意力对齐*(少量、甚至 4B 上大量 BP 都装不进)。

自然的反驳:这是 ZeroBP 的属性、BP 数量的属性、还是*骨架*(冻结 reservoir 注意力 + 末位塌缩)的属性?这个问题正是对照线要回答的。

## 5. 对照线:Phase H,一个可训练注意力骨架

### 5.1 设计与隔离

Phase H(一个"v3.0"研究栈,由 ADR-005 与专用 charter 治理)是一个干净的切换:一个标准 pre-LN Transformer,多头**双向**自注意力、GELU-MLP 块、残差连接,以及一个**不塌缩的 mean-pool 读出**,用**普通全反向传播(AdamW)**训练。它刻意*不*聪明 —— 目的是用一个常规骨架作架构对照。它**严格隔离**:`phase_h/` 零依赖 ZeroBP 训练器(纯 PyTorch,可迁出独立仓库),提交永不 import 它,且每次改动后重验提交默认路径(`final_ppl 6.251`、零 autograd)。

在与 §4 *相同*的合成分布上评测,加真实 SNLI、真实 GSM8K、真实 SST-2,采用同样的公平读出纪律。

### 5.2 G0 —— 骨架是不是限制?(合成)

一个 0.80M 参数、4 层、4 头的 Phase-H 模型,全 BP 在*相同*合成任务上:

| 任务(合成,同 §4 分布) | Phase H | ZeroBP-4B / 小(§4) |
|---|---|---|
| NLI(3 类) | **100%** @第 500 步 | 深 BP 65.7%(小)/ chance(4B) |
| 两步算术(5 类) | **100%** @第 1000 步 | *每个* BP 深度 chance |

一个 0.8M 标准可训练注意力模型装进了 ZeroBP 装不进的*两种*结构 —— 强证据表明 §4 边界是**架构性的,不是任务内在的**。

### 5.3 G1 —— 真实关系数据(SNLI)

规模化到一个 12.4M 参数(6 层、8 头、d=256)Phase-H 模型,全 BP 在**真实 SNLI**(549k 训练 / 9.8k 验证,12.9k 步)上:**val 准确率 = 69.97%**(majority 33.8%)。ZeroBP-4B 在 NLI 上是 **chance(33.4%)**。新骨架在一个真实关系 benchmark 上达到现代小 LLM 区间,而 ZeroBP 栈结构上做不到。(因 Phase H 是独立栈,这*不*改任何锁定的 ZeroBP 结论;它*语境化*了后者。)

### 5.4 G2 —— 多步能多深?

对算术 fold 扫描顺序步数 k,逐深度训练一个全新 Phase-H 模型,两个规模:

| n_steps | v1: 6L×8H d256, 4.7M, 6k 步 | deep: 12L×8H d384, 21.3M, 15k 步 |
|---|---|---|
| k = 2 | **100%** | **100%** |
| k = 3 | 100%(2.67M,本地) | — |
| k = 4 | 22.5% | 21.3% |
| k = 6 | 19.9% | 19.8% |
| k = 8 | 20.6% | 21.4% |

(chance = 20%)。Phase H 装进**浅多步(k≤3=100%)**,决定性超越 ZeroBP(任何深度 chance,连 k=2 都不过)。但 k≥4 是一堵**墙**:**5× 参数 + 2.5× 步数一点没动**。这*推翻了该"欠容量"假设* —— k≥4 失败在此设定下不由规模解决。如实标注设定:任务是*直接答案分类*,mean-pool 读出,**无 curriculum、无 chain-of-thought** —— 最可能打破 k≥4 墙的两条机制,本文未测。

### 5.5 G2b 与 SST-2 —— 诚实上限与一个读出坍缩教训

- **G2b —— 生成式 GSM8K(stretch)。** 一个 4.87M 参数 causal char-LM(6 层),训练*生成*答案 token 并按 exact-match 评分,在真实 GSM8K 上达 **2.0%**。已提前标注为 stretch(小从零 char LM 缺乏自然语言 GSM8K 的规模/预训练;*合成*生成式 smoke 达 66.6%)。如实报告而非包装:机器工作,模型不行。
- **SST-2 —— 一个纠正后的负结果。** ZeroBP-4B 栈在真实 GLUE/SST-2 上先返回*逐位相同*的 49.08%(zero-shot / emb-BP / emb+attn-BP)。诊断重跑揭示原因:**闭式读出头坍缩**(872 个样本全预测 class-0,`[872,0]`),故 49.08% 是"总猜负",一个 artifact。公平 **BP 线性探针**(用两个类)揭示真值:zero-shot **51.5%** → emb **52.5%** → emb+attn **53.3%**(majority 50.9%,真值 `[428,444]`)。故 ZeroBP-4B 在*真实*情感上 **≈ chance**(+0.6…+2.4pp over majority)—— 远离合成 bag 情感的 79%。**真实词汇情感与 NLI/多步同属"装不进"一类**;合成 79% 是 embedding 可分的特例,不代表一般情感能力。这是读出选择制造表面数字的第二处,强化了公平读出纪律。

### 5.6 最终记分卡

| 维度 | Phase H(新骨架) | ZeroBP-4B(锁定) | 结论 |
|---|---|---|---|
| 合成 NLI + 算术(G0) | 100% / 100% | 65.7%(小)/ chance | 骨架是限制 |
| 真实 SNLI(G1) | **69.97%** | 33.4%(chance) | 关系结构在新栈可安装 |
| 多步深度(G2) | k≤3=100%, k≥4 抗规模墙 | 任何深度 chance | 新栈有自身天花板 |
| 生成式 GSM8K(G2b) | EM ~2% | — | 小模型 stretch 失败 |
| 真实 SST-2 | ≈ chance(探针 53%) | ≈ chance | 真实情感 ≫ 合成 bag;ZeroBP 边界在真实数据上确认 |

## 6. 讨论:算法 vs 架构 vs 任务

由于每次只改一个变量,三种解读得以分离:

**ZeroBP-4B 天花板是架构性的。** 关系几何不在冻结表示的每个位置(§4.4),故无读出可恢复;可训练 attention 单独无用(§4.4);多步在任何 BP 深度都死(§4.4)。头号架构疑点正是 §3.2 的两个冻结结构选择:一个**冻结随机注意力 reservoir**(无法形成跨句对齐)和一个**末位塌缩**(在路由块之前丢弃序列结构)。embedding BP 只能装入 token 级*线性可组合*的东西 —— 故 bag 式情感(79%)成立而关系对齐 / 顺序计算不成立。

**BP 必要但不充分 —— 它需要一个可训练的结构底座。** 一点 BP 装进 bag 结构但不装关系/多步结构*在此骨架上*;同样的任务在一个可训练注意力骨架上被全 BP 轻易装入(§5.2)。故梯度只在架构给结构留出可训练位置时有用。这把"ZeroBP 做不了 NLI"重述为"**任何 BP 注入一个冻结 reservoir/末位塌缩骨架都装不进关系结构;一个标准注意力骨架轻松做到。**"

**任务结构决定难度序,连好骨架也有诚实上限。** 两栈上的难度序一致 —— bag ≪ 关系 ≪ 多步深度 ≪ 真实生成式数学。Phase H 跨过关系边界与浅多步,但它的 **k≥4 深度墙抗规模**(§5.4),提示深层顺序推理需要的*不只是骨架 + 规模* —— 很可能是 curriculum、中间监督、或 chain-of-thought(显式生成中间运行值),这些本文都未测。生成式-GSM8K 与真实-SST-2 结果进一步给论断划界:在受控任务上跨过一道边界,不等于一般能力。

**方法论要点。** 本研究里两个表面结果是*读出*的假象、不是表示的事实(§4.2 的 57% 情感 SGD 头混淆,与 §5.5 的 49% SST-2 头坍缩)。关于表示能力的论断应总是通过一个**公平、任务无关的探针**,最好*两个*(闭式与 BP 线性),它们在其中一个退化时恰好互相矛盾。

## 7. 结论与未来工作

本报告呈现了一项三线边界研究。**ZeroBP-4B** 是一个可复现、确定性、无反向传播的 4B 语言模型,在单 T4 预算下对 LM 与 bag 式任务确实有用(WikiText-103 ppl ≈ 1355–1391;情感 79%),并作为测量能力边界的可控对象。这些边界 —— 关系与多步失败 —— 被一个严格隔离的 **Phase H** 对照证明是**架构性的**:一个标准可训练注意力 Transformer + 全 BP 把同样的合成任务解到 100%、在真实 SNLI 上达 **69.97%**(vs chance)、并装进浅多步推理 —— 同时暴露它*自己*的诚实上限:k≥4 抗规模多步深度墙、真实 GSM8K ~2%、以及也困扰 ZeroBP 的近 chance 真实 SST-2。

**角色。** ZeroBP-4B:一个资源受限预训练 + 边界测量工具,在 LM/简单组合上有效,在关系/多步上硬限。Phase H:一个现代骨架基线,证明 NLI 与浅多步*可学*(不是任务内在壁垒),并浮现一个新的推理深度研究问题。

**未来工作。** (i) 用 curriculum / chain-of-thought / 中间监督(而非纯规模)打破 k≥4 墙;(ii) 测试*先进*的 ZeroBP 局部规则在一个*可训练注意力*底座上能否不用全 BP 恢复部分 Phase-H 能力(两条线合流);(iii) 把 Phase H 规模化(数据 + 参数,MNLI/GSM8K)向关系/推理维度上的现代小 LLM 行为逼近;(iv) 在保持零 autograd 纯净下,用更好的局部规则提升 ZeroBP-4B 的 LM 质量。
